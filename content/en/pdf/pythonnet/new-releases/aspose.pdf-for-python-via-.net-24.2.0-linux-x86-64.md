Developers using Python on Linux can now leverage the enhanced capabilities of Aspose.PDF for Python via .NET 24.2.0. This release brings a collection of new features and improvements to augment their existing Python PDF processing apps or create new ones from scratch. 

### New Features

We have introduced a number of new features in this release:
- Effortlessly retrieve vector data from your PDF documents.
- You can now load DjVu and CDR file formats within your Python PDF processing apps.
- Add page reference in the callback for HOCR text recognition on Linux machines.
- Using version 24.2.0 of Aspose.PDF for Python via .NET, you can pass a page in the form of an image to the HOCR callback method.
- Convert PDF to Markdown with enhancements to image usage.
- Transform searchable PDF files to XPS format, having the ability to select and copy text within the resulting XPS document.

### Feature Enhancements

We have added feature enhancements in this release:
- Accurately extract images in PDF to Markdown conversion.
- Manage page settings while converting HTML to PDF.
- Review comments' status precisely in `TextAnnotation` on Linux systems.
- Enhanced the functionality of the document repair API.
- Control the LaTeX document compilation with version 24.2.0 of the Python PDF API.
- CSS styling and `PageInfo` are supported in Markdown loading capabilities.

### Bug Fixes

This release also addresses numerous bugs reported in previous versions, including issues with: 
- Text extraction and rendering.
- XPS to PDF conversion.
- Splitting PDF documents.
- Handling absolute image positions in HTML to PDF conversion
- PDF to PDFA1A conversion.

### Public API and Backward Incompatible Changes

### Added APIs

We have added several new classes, methods, and properties to the public API in this release. Some of these are:
- aspose.pdf.annotations.MarkupAnnotation.clear_state
- aspose.pdf.annotations.MarkupAnnotation.set_review_state(aspose.pdf.annotations.AnnotationState, str)
- aspose.pdf.annotations.MarkupAnnotation.set_marked_state(bool)
- aspose.pdf.CdrLoadOptions
- aspose.pdf.CdrLoadOptions.#ctor
- aspose.pdf.DjvuLoadOptions
- aspose.pdf.DjvuLoadOptions.#ctor
- aspose.pdf.DocSaveOptions.re_save_fonts bool
- aspose.pdf.LoadFormat.DJVU
- aspose.pdf.LoadFormat.CDR
- aspose.pdf.printing.PaperSource.#ctor(aspose.pdf.printing.PaperSourceKind, str)

### Removed APIs

We have removed the following property from the public API in this release:
- aspose.pdf.annotations.TextAnnotation.state
