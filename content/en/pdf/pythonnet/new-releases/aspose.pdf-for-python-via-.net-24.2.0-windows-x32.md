Aspose.PDF for Python via .NET 24.2.0 is here to boost your Python development experience on Windows 32-bit systems. This release introduces valuable new features and bug fixes for handling PDF documents.

### New Features

We have introduced a number of new features in this release:
- Seamlessly extract vector data from PDF files.
- Work with DjVu and CDR format files within your Python PDF processing apps.
- Easily provide a callback for HOCR text recognition with page reference.
- With Aspose.PDF for Python via .NET 24.2.0,you can pass a page as an image to HOCR callback method.
- PDF to Markdown conversion with image usage enhancements.
- Converting searchable PDF documents to XPS with the ability to select and copy text within the resulting XPS document preserved.

### Feature Enhancements

We have added feature enhancements in this release:
- Image extraction in PDF to Markdown conversion with better accuracy.
- Control the page settings during HTML to PDF conversion.
- Precisely review comments' status in `TextAnnotation` on Windows 32-bit machines.
- Optimized document repair API functionality.
- Fetch property values directly from Action Script code embedded within your PDF documents.
- Manage the LaTeX document compilation with version 24.2.0 of the Python PDF API.
- Markdown loading capabilities are extended by supporting CSS styling and `PageInfo`.

### Bug Fixes

This release also addresses numerous bugs reported in previous versions, including issues with: 
- Splitting PDF documents.
- XPS to PDF conversion.
- Handling absolute image positions in HTML to PDF conversion.
- Text extraction and rendering.
- PDF to PDFA1A conversion.

### Public API and Backward Incompatible Changes

### Added APIs

We have added several new classes, methods, and properties to the public API in this release. Some of these are:
- aspose.pdf.annotations.MarkupAnnotation.clear_state
- aspose.pdf.annotations.MarkupAnnotation.set_review_state(aspose.pdf.annotations.AnnotationState, str)
- aspose.pdf.annotations.MarkupAnnotation.set_marked_state(bool)
- aspose.pdf.CdrLoadOptions
- aspose.pdf.CdrLoadOptions.#ctor
- aspose.pdf.DjvuLoadOptions
- aspose.pdf.DjvuLoadOptions.#ctor
- aspose.pdf.DocSaveOptions.re_save_fonts bool
- aspose.pdf.LoadFormat.DJVU
- aspose.pdf.LoadFormat.CDR
- aspose.pdf.printing.PaperSource.#ctor(aspose.pdf.printing.PaperSourceKind, str)

### Removed APIs

We have removed the following property from the public API in this release:
- aspose.pdf.annotations.TextAnnotation.state
