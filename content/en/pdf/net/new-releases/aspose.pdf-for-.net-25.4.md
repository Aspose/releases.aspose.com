#### Release Highlights

Aspose.PDF for .NET 25.4 introduces automatic tagging for PDF/A conversion, integrated OCR with AI copilot, bulk document printing, and custom encryption handler support.

### ‚ú≥Ô∏è New Feature

- **Auto-Tagging in PDF/A Conversion**: Automatically generate logical document structures during PDF to PDF/A conversion with `AutoTaggingSettings`.
- **OCR via AI Copilot**: Extract text from scanned PDFs and images using `OpenAIOcrCopilot` integrated with GPT-4 vision model.
- **Multi-Document Printing**: Print multiple documents in a single job using `PdfViewer.PrintDocuments()`.
- **Custom Encryption Support**: Create your own security handler via `ICustomSecurityHandler` interface for full encryption control.

### üîß Enhancement

- **PDF/A-4 Support**: Added new compliance level to `PdfAConverter` plugin.
- **Certificate Chain Checking**: Enhanced signature validation now supports revocation chain validation.
- **New Annotation APIs**: Added setter for annotation creation date.

### üõ† Fix

- Fixed PDF/A validation failures, formatting errors with Japanese text, SVG rendering, encryption metadata issues, and DOCX output anomalies.
- Resolved over 30 PDF format conversion, rendering, and metadata inconsistencies.

### üîç Public API and Backward Incompatible Changes

**Added APIs**:
- Dozens of new types under `Aspose.Pdf.AI`, `AutoTaggingSettings`, and `ICustomSecurityHandler`.
- `PdfViewer.PrintDocuments()` overloads for various input types.

**Removed APIs**:
- Deprecated fields and types, including `PdfToMarkdown.HeadingLevels`, and `OptimizationOptions.LinkDuplicateStreamsScanLevel`.
