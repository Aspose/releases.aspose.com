#### Release Highlights

Aspose.OCR for .NET 24.11.0 (DLLs only) enables new language support for Japanese and Korean, enhances PDF output with custom fonts, and speeds up document-wide text extraction.

### ✳️ New Feature

- **Japanese & Korean OCR Support**: Added `Language.Jpn` and `Language.Kor` options in the public API. Supports **mixed-language OCR** for real-world documents. *(Requires optional OCR feature modules to be installed.)*

### 🔧 Enhancement

- **Better Searchable PDF Output**:
  - Improved text layer generation
  - Enhanced compatibility with embedded fonts
- **Performance Optimizations**: Recognition is now **faster and more precise** across various document layouts and image types.

### 🛠 Fix

- *(Improvements integrated via enhancements; no standalone bug fixes noted.)*

### ⚠️ Deprecation Notice

These APIs are **deprecated** and scheduled for removal in **25.1.0 (January 2025)**:
- `DetectAreasMode` values: `NONE`, `TEXT_IN_WILD`, `PHOTO`, `COMBINE`, `DOCUMENT`
  - 👉 Replace with `LEAN`, `UNIVERSAL`, or `MULTICOLUMN`
- `Country.UNIVERSAL` → use `Country.NONE`
- Non-ISO compliant `Language` values → migrate to ISO 639-2 compliant constants

> Legacy code will continue to work for now but **must be migrated** before January 2025.
