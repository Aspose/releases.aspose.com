#### Release Highlights

Aspose.OCR for .NET 24.10.0 introduces advanced mixed-language OCR, modern layout detection modes, and improved support for TIFF files.

### ✳️ New Feature

- **Mixed-Language OCR**: Added support for recognizing **Chinese-English hybrid content**, enabling seamless recognition in multilingual documents.

### 🔧 Enhancement

- **Improved Chinese OCR Accuracy**: Significantly boosted character recognition for Chinese scripts, particularly in modern and simplified text.
- **New Layout Detection Modes**: Introduced three modern `DetectAreasMode` settings:
  - `UNIVERSAL`: Best for irregular content and photos.
  - `MULTICOLUMN`: Optimized for articles and documents with **multi-column** formatting.
  - `LEAN`: Prioritizes speed for simple documents with minimal layout complexity.
- **TIFF Compatibility**: Enhanced internal decoding to improve **TIFF rendering reliability and OCR accuracy**.

### 🛠 Fix

- *(No direct bug fixes, all updates are enhancements.)*

### ⚠️ Deprecation Notice

- **Language Codes**: Deprecated legacy recognition language codes in favor of **ISO 639-2** standards (as of 24.3.0).
- **Passport Recognition Mode**: Deprecated `Country.UNIVERSAL`; use `Country.NONE` instead to disable MRZ data extraction (since 24.6.0).
- **DetectAreasMode Values to Be Removed in 25.1.0**:
  - `NONE`: Use `LEAN` instead.
  - `TEXT_IN_WILD`, `PHOTO`, `COMBINE`: Use `UNIVERSAL` instead.
  - `DOCUMENT`: Use `MULTICOLUMN`.

> These values will be removed in **Aspose.OCR 25.1.0 (January 2025)**. Legacy code will work but should be migrated now.
