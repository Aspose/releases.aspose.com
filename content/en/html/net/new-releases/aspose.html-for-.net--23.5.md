## Release Summary

The new features and bug fixes in Aspose.HTML for .NET 23.5 make it a more powerful and reliable tool for C#, .NET App developers. The new API methods make it easier to customize the behavior of the Aspose.HTML API, and the bug fixes improve the accuracy and consistency of HTML parsing and rendering.

## HTML API Supports `WOFF2` Font Format

Aspose.HTML for .NET now supports the `WOFF2` font format, which is the latest and most efficient web font format. This allows developers to create C#, .NET Apps with smaller file sizes and faster loading times.

## Improved `TTF` Font Support

Aspose.HTML for .NET now provides better support for the `TTF` font format. This can lead to more accurate and consistent rendering of `TTF` fonts in C#, .NET Apps.

## Improved HTML parsing and rendering

Aspose.HTML for .NET has been optimized to improve the quality of HTML parsing and rendering. This can lead to more accurate and consistent results when converting HTML documents to other formats.

## Better Arabic Text Conversion Support

Fixed a bug that caused the conversion from `HTML` to `PNG` to fail for Arabic text.

## Better `EPUB` to `PDF` Conversion Support

Fixed a bug that caused only the first page of an `EPUB` document to be converted to `PDF` in some cases.

> For a complete list of features, enhancements, and bug fixes in this release please visit, [Aspose.HTML for .NET 23.5 Release Notes](https://releases.aspose.com/html/net/release-notes/2023/aspose-html-for-net-23-5-release-notes/).
