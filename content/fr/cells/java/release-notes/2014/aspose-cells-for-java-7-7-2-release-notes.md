{{% alert color="primary" %}} 

 Cette page contient des notes de version pour[Aspose.Cells for Java 7.7.2](https://releases.aspose.com/cells/java/new-releases/aspose.cells-for-java-7.7.2/)

{{% /alert %}} 

 Aspose.Cells for Java a été mis à jour vers la version 7.7.2 et nous sommes heureux d'annoncer que cette version apporte l'ajout d'environ 20 nouvelles améliorations utiles.
En utilisant Aspose.Cells for Java, vous pouvez travailler avec XLS, SpreadsheetML, OOXML, XLSB, CSV, HTML, ODS, PDF, XPS et d'autres formats dans vos applications. Vous pouvez également générer, modifier, convertir, rendre et imprimer des classeurs sans utiliser Microsoft Excel.
Consultez la documentation pour savoir comment démarrer avec Aspose.Cells for Java.
Notez que ce téléchargement contient une version entièrement fonctionnelle du produit, mais sans jeu de licences, il fonctionnera en mode d'évaluation avec certaines limitations. Pour tester Aspose.Cells sans ces limitations d'évaluation, vous pouvez demander une licence temporaire gratuite de 30 jours.
Voici une liste des modifications apportées à cette version de Aspose.Cells for Java.

Principales caractéristiques
Prise en charge de l'ouverture de fichiers MS Excel 2013 protégés par mot de passe

Autres améliorations et modifications

Nouvelles fonctionnalités

(CELLSJAVA-40475) - Imprimer des commentaires sur un document Excel
(CELLSJAVA-40732) - Prise en charge du paramètre d'alignement distribué justifié

Améliorations

(CELLSJAVA-40719) - Suppression de l'utilisation du code dur de com.sun.image.codec.jpeg.JPEGCodec
(CELLSJAVA-40708) - Afficher la ligne avec la hauteur précédemment attribuée

Insectes

(CELLSJAVA-40711) - Axis.setTickLabelSpacing() ne fonctionne pas pour la version XLS
(CELLSJAVA-40714) - Cell bordures étaient rendues sur des cellules opaques
(CELLSJAVA-40710) - Inférieur à l'erreur de calcul
(CELLSJAVA-40715) - Problème d'analyse des caractères génériques dans le calcul de la formule
(CELLSJAVA-40707) - Certaines colonnes sont étroites pour contenir correctement le texte
(CELLSJAVA-40720) - IStreamProvider.setCustomPath n'affecte pas le fichier tabstrip.htm
(CELLSJAVA-40729) - Extraction d'un problème de date chinoise formatée
(CELLSJAVA-40728) - getStringValue() renvoie #VALEUR !
(CELLSJAVA-40722) - La validation ne fonctionne pas après la conversion au format XLS
(CELLSJAVA-40690) - XLSX à PDF : le soulignement sous l'hyperlien s'étend aux positions de caractères adjacentes

Des exceptions

(CELLSJAVA-40629) - Exception lors de l'ouverture de xlsx converti en xls
(CELLSJAVA-40630) - Exception lors de l'ouverture de xlsx converti en xls
(CELLSJAVA-40717) - CellsException : nom de cellule non valide sur le constructeur de classeur
(CELLSJAVA-40723) - Rendu com.aspose.cells.CellsException : java.lang.String ne peut pas être converti en java.lang.Double
(CELLSJAVA-40724) - com.aspose.cells.CellsException sur SheetRender
(CELLSJAVA-40727) - Exception : java.lang.NullPointerException dans le rendu des graphiques dans une feuille de calcul

Noter
Étant donné que la base de code de Aspose.Cells for Java correspond au code de la version .NET pertinente, la plupart des modifications, améliorations et correctifs inclus dans le Aspose.Cells for .NET v7.7.2 sont également inclus dans ce Aspose.Cells for Java v7.7.2.
