Öffentliche API und rückwärts inkompatible Änderungen

Diese Version hat zwei große Änderungen API:

  • Das Animations system wurde neu erstellt, sodass wir einige Namen für die zukünftige Verwendung reservieren können, wenn die Formate CAD unterstützt werden. -Diese Version umbenanntKurveZuSchlüssel rahmenS equenz, UndCurve MappingZuBind Point. Die veralteten Schnitts tellen werden von Aspose.3D for .NET 20.03 entfernt. Methoden, die diese Klassen verwenden, bieten den Ersatz als Alternative. -Während die alten Namen in 19.12 noch vorhanden sind, benötigt der Code, der auf diesen Änderungen beruht, weniger oder sogar keine Änderungen (Wenn Sie Typ infer verwenden).
  • Entfernt den älteren OpenGL-Renderer und überarbeitet den Renderer, damit er am besten mit dem zugrunde liegenden Vulkan-Treiber funktioniert. Low-Level-Schnitts tellen wurden geändert, während die High-Level-Rendering-Schnitts tellen intakt blieben. -Der refactorierte Renderer verfügt über eine bessere Rendering-Leistung mit mehr Flexibilität und Erweiterbar keit. -Die Render methode in derSzeneKlasse hat keine Änderungen. Wenn Sie eine High-Level-Rendering-Schnitts telle verwenden, müssen Sie nichts ändern. -Die Low-Level-API hat eine bahnbrechende Änderung. Möglicher weise müssen Sie sich an den Support für die Code-Migration wenden.
